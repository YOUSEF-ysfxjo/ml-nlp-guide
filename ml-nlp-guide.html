From Linear Models to Word Embeddings — A Complete Breakdown

1. The Linear Model

The simplest model in all of machine learning is:

y = Wx + b

What Each Symbol Means

SymbolNameDescriptionExample
xInput / FeaturesThe raw data you feed inx = [word_count, sentence_length]
WWeightsHow much influence each feature hasA matrix of numbers the model learns
bBiasShifts the output up or downAllows the model to fit data not centered at the origin
yOutputThe predictionA class label, a score, etc.

What Does It Do Geometrically?

In 2D space, a linear model draws a straight line to separate two classes:

ax + by + c = 0

In 3D, it draws a plane. In higher dimensions, it draws a hyperplane.

Concrete Example

Imagine classifying an email as spam or not-spam:

x = [number_of_links, number_of_caps, message_length]
W = [0.8, 0.6, -0.1]     ← learned weights
b = -2.0                   ← learned bias

y = (0.8 × links) + (0.6 × caps) + (-0.1 × length) - 2.0

If y > 0 → Spam. If y < 0 → Not spam.

2. The Problem with Linear Models

Linear models are powerful but have a fundamental limitation:

✅ They can learn linear relationships

❌ They cannot learn non-linear relationships

The XOR Problem — A Classic Example

Consider this truth table:

Input AInput BOutput (A XOR B)
000
011
101
110

If you plot these 4 points, no single straight line can separate the 1s from the 0s. The pattern is inherently non-linear.

  B
  |
1 |  ○  ✕
  |
0 |  ✕  ○
  +--------A
     0    1

○ = Output 0
✕ = Output 1
→ No straight line can separate them!

This means: A single linear model cannot solve XOR. This was actually a major crisis in early AI, famously described by Minsky & Papert in 1969.

3. Neural Networks

The solution: stack multiple layers to build a network that can model complex, non-linear patterns.

Architecture

Input Layer
    ↓
Hidden Layer 1  ← learns low-level features
    ↓
Hidden Layer 2  ← learns high-level features
    ↓
Output Layer    ← makes the final prediction

What Happens in Each Layer?

Every layer performs two operations:

Step 1 — Linear transformation:

z = Wx + b

Step 2 — Activation:

a = f(z)

Where f is an activation function (more on this below).

A Concrete Example

Say you're building a network to recognize handwritten digits (0–9):

  • Layer 1 might detect edges and curves
  • Layer 2 might detect shapes like loops or lines
  • Layer 3 might combine those into digit patterns
  • Output picks the most likely digit

Each layer transforms the data into a new, more useful representation.

4. Activation Functions

Why Are They Necessary?

Imagine a 2-layer network without activation functions:

Layer 1:  z₁ = W₁x + b₁
Layer 2:  z₂ = W₂z₁ + b₂

Substitute Layer 1 into Layer 2:

z₂ = W₂(W₁x + b₁) + b₂
   = (W₂W₁)x + (W₂b₁ + b₂)
   = W_newx + b_new

You just collapsed it into a single linear model. No matter how many layers you add, without activation functions, the entire network is equivalent to one linear layer.

Common Activation Functions

ReLU (Rectified Linear Unit)

f(z) = max(0, z)
  • If z = -5 → output is 0; if z = 3 → output is 3
  • Fast to compute, works great in deep networks
  • Most commonly used today

Sigmoid

f(z) = 1 / (1 + e^(-z))
  • Outputs values between 0 and 1
  • Great for binary classification
  • Problem: "vanishing gradients" in deep networks

Tanh (Hyperbolic Tangent)

f(z) = (e^z - e^(-z)) / (e^z + e^(-z))
  • Outputs values between -1 and 1
  • Zero-centered (better than sigmoid for hidden layers)
  • Also suffers from vanishing gradients

5. Learnable Parameters

What Does a Model Actually Learn?

A neural network learns its parameters:

  • Weights (W) — how strongly each input influences each output
  • Biases (b) — a constant offset that shifts the decision boundary

These are called Learnable Parameters because they start as random values and are gradually improved through training.

How Many Parameters Does a Model Have?

For a single layer with input size 784 (e.g., 28×28 pixel image) and output size 256 neurons:

Weights: 784 × 256 = 200,704
Biases:  256
Total:   200,960 parameters

Modern LLMs (like GPT-4) have hundreds of billions of parameters.

6. Optimization — How Networks Learn

The Core Idea

Learning = finding the values of W and b that make predictions as accurate as possible. This is done by minimizing a Loss Function L, which measures how wrong the model is.

Loss Functions (Examples)

TaskCommon Loss Function
Binary ClassificationBinary Cross-Entropy
Multi-class ClassificationCategorical Cross-Entropy
RegressionMean Squared Error (MSE)

Gradient Descent

θ = θ - η∇L

Where: θ (theta) = parameters (W and b); η (eta) = learning rate; ∇L = gradient.

Intuition: Imagine you're blindfolded on a hilly terrain and want to find the lowest valley. Gradient descent tells you which direction is "downhill" and takes a step in that direction.

Popular Optimization Algorithms

AlgorithmDescriptionAdvantage
Gradient DescentComputes gradient over entire datasetStable but slow
SGDOne sample at a timeFast but noisy
AdamAdaptive learning rates per parameterBest of both worlds; most popular
RMSPropMaintains moving average of gradientsGood for RNNs

Backpropagation — the chain rule applied backwards through the network to calculate how much each weight contributed to the error.

7. The Text Problem

Neural networks are mathematical machines. They only understand numbers. But language is:

"I love artificial intelligence"

This is not a number. We need a way to represent text as numbers while preserving meaning.

8. One-Hot Encoding

How It Works

  1. Build a vocabulary (list of all unique words)
  2. Assign each word an index
  3. Represent each word as a vector of all zeros, with a 1 at that word's index

Example: Vocabulary ["I", "love", "AI", "hate", "cats"] → "I" = [1,0,0,0,0], "love" = [0,1,0,0,0], etc.

Problems with One-Hot Encoding

Curse of Dimensionality: Real vocabularies are massive — each word vector can be 100,000 dimensions. No Semantic Meaning: The distance between "king" and "queen" is the same as "king" and "car". No Relationship Between Words: The model can't know that "big" and "large" mean similar things.

9. Word Embeddings

Instead of huge, sparse one-hot vectors, map each word to a small, dense vector that captures meaning. Semantically similar words end up close together in vector space.

Famous Analogy: Word Arithmetic

king - man + woman ≈ queen
Paris - France + Italy ≈ Rome
walking - walk + swim ≈ swimming

The model has learned real semantic structure of language — without ever being explicitly taught grammar or word meanings.

10. Word2Vec

Word2Vec (2013, by Mikolov et al. at Google) is a simple neural network that learns word embeddings by training on a massive text corpus.

Words that appear in similar contexts have similar meanings.

Two Training Approaches

CBOW — Continuous Bag of Words: Predict the target word from its surrounding context. Skip-Gram: Predict the surrounding context from a single word. Skip-gram tends to work better for rare words; CBOW is faster to train.

11. Word2Vec Architecture

Pipeline: Input word (One-Hot) → Embedding Matrix W (V×D) → Hidden vector h → Output Matrix W' → Softmax → Probability over all words.

The rows of W are the word embeddings we care about. Multiplying a one-hot vector by a matrix is equivalent to looking up a row — this is why embedding layers are implemented as efficient lookup tables.

12. Dot Product Similarity

The model computes similarity between hidden vector h and each output word vector using the dot product: u_j = h · w'_j. The dot product is large when two vectors point in the same direction. Softmax converts scores into a probability distribution over the vocabulary.

13. Distributional Hypothesis

"You shall know a word by the company it keeps." — J.R. Firth, 1957

Words that appear in similar contexts have similar meanings. After training on a large corpus, Word2Vec automatically discovers countries→capitals, gender relationships, verb tenses, and comparative adjectives — all from raw text.

14. Limitations of Word2Vec

Static Representations (Polysemy): Each word gets one fixed vector — "bank" (financial vs river) gets the same vector. Rare Words Learn Poorly: Few gradient updates → weak embedding. Out-of-Vocabulary Words: Words not seen during training have no embedding. (Partially solved by FastText and BPE.)

15. Representing Sentences with Word2Vec

Method 1 — Averaging: Sentence vector = mean of word vectors. Problem: loses word order ("Dog bites man" vs "Man bites dog" → same vector). Method 2 — Concatenation: Vector length depends on sentence length; networks need fixed-size inputs. Neither method captures word order, long-range dependencies, or dynamic context.

16. The Road Ahead — NLP Evolution Map

Word2Vec (2013) → RNNs → LSTM/GRU → Attention (2015) → Transformer (2017) → BERT, GPT, T5 (2018+) → LLMs (2020s).

— Related Papers

Summaries, structure, and visuals for papers linked to this guide

Efficient Estimation of Word Representations in Vector Space

Mikolov et al. 2013 Google / ICLR
PDF arXiv Code (GitHub)

Summary

The paper introduces two neural network architectures for learning distributed word representations from large text corpora: CBOW (Continuous Bag of Words), which predicts a target word from its context, and Skip-gram, which does the opposite—predicting context words from a single target word. Both produce dense, low-dimensional vectors that capture semantic similarity and linear analogies (e.g. king − man + woman ≈ queen). The models are trained efficiently without dense matrix multiplications by using a hierarchical softmax (and later, negative sampling). Experiments on word similarity and analogy tasks show large gains over previous count-based and neural approaches.

Key points

  • Distributional hypothesis in practice: Words in similar contexts get similar vectors.
  • CBOW: Input = sum/average of context word vectors → output = predicted center word (faster, good for frequent words).
  • Skip-gram: Input = center word → output = predicted context words (better for rare words, often preferred).
  • Hierarchical softmax: Replaces the full V-way softmax with a binary tree to reduce complexity from O(V) to O(log V).
  • Linear structure: Word relations (e.g. gender, tense) appear as roughly linear offsets in vector space.
  • Scale matters: Larger corpus and dimensions improve quality; 300–600 dimensions work well.

Structure

  • → Abstract & intro: motivation for dense representations
  • → Previous work: NNLM, RNNLM, latent representations
  • → New models: CBOW and Skip-gram objectives and training
  • → Complexity: comparison with NNLM, hierarchical softmax
  • → Experiments: similarity, analogy, NCE/negative sampling
  • → Conclusion: simple models, large data, linear structure

Visual — CBOW vs Skip-gram

Pipeline: One-hot → embedding matrix W (V×D) → hidden vector h → output matrix W′ → Softmax (or hierarchical softmax) → probability over vocabulary.

Try it: Gensim (Word2Vec in Python)

Same algorithm as the paper; run in Python with a small corpus to see embeddings and similarity.

from gensim.models import Word2Vec

sentences = [
    ["the", "cat", "sat", "on", "the", "mat", "."],
    ["the", "dog", "sat", "on", "the", "mat", "."],
    ["cats", "and", "dogs", "are", "animals", "."],
    ["the", "cat", "and", "the", "dog", "played", "."],
]
model = Word2Vec(sentences, vector_size=8, window=2, min_count=1,
                 sg=1, negative=3, epochs=5, alpha=0.025, seed=42)
wv = model.wv
print("Vocab size:", len(wv))
print("Embedding for 'cat':", wv["cat"])
print("Cosine sim(cat, dog):", wv.similarity("cat", "dog"))
print("Most similar to 'cat':", wv.most_similar("cat", topn=3))

Expected output (example)

Vocab size: 13
Embedding for 'cat': [ 0.01 -0.09  0.06  0.05  0.11  0.06 -0.03  0.12 ]
Embedding for 'dog': [ 0.07  0.04 -0.02  0.08  0.01 -0.01 -0.01 -0.07 ]
Cosine sim(cat, dog): -0.26
Most similar to 'cat': [('.', 0.69), ('and', 0.67), ('mat', 0.40)]

Explanation of the results

  • Vocab size (13): Number of unique words in the corpus. With only 4 sentences, the vocabulary is small.
  • Embedding for 'cat' / 'dog': Each word is a vector of 8 floats (vector_size=8). The model learned these numbers so that words in similar contexts end up with similar vectors.
  • Cosine sim(cat, dog): Measures how “aligned” the two vectors are (−1 to 1). Here it can be negative or low because the corpus is tiny; with a large corpus, cat and dog usually get positive similarity (e.g. 0.5–0.8).
  • Most similar to 'cat': Words whose vectors are closest to cat’s. On this tiny corpus you may see “.”, “and”, “mat” (same sentence context); on a large corpus you’d see “dog”, “kitten”, “animal”, etc.

Full walkthrough and from-scratch implementation: see word2vec_from_scratch.ipynb in this project.

Distributed Representations of Words and Phrases and their Compositionality

Mikolov et al. 2013 Google / NeurIPS
PDF arXiv Code (GitHub)

Summary

This follow-up extends the Skip-gram model from the first paper with three main ideas: negative sampling (train on a few random “negative” words instead of the full vocabulary, making training much faster), subsampling of frequent words (randomly drop very common words like “the” so they don’t dominate and rare words get more updates), and phrase representations (merge frequent bigrams into single tokens, e.g. “New_York”, so that multi-word expressions get their own vectors). The paper shows that Skip-gram with negative sampling outperforms hierarchical softmax on analogy tasks and that phrase-aware training improves compositionality and similarity.

Key points

  • Negative sampling: For each (center, context) pair, maximize probability of the true context and minimize probability of k random “negative” words. Replaces full softmax with a simple binary classification; much faster and often better.
  • Subsampling: Discard word w with probability 1 − √(t/f(w)) where f(w) is frequency; reduces dominance of “the”, “a”, etc. and improves quality for rare words.
  • Phrases: Score bigrams by co-occurrence, merge top ones into single tokens (e.g. “ice_cream”), then train Skip-gram on the merged corpus so phrases get dedicated vectors.
  • Additive compositionality: Word vectors can be added to approximate phrase/sentence meaning (e.g. vector(“Germany”) + vector(“capital”) ≈ vector(“Berlin”)); phrase training refines this.

Structure

  • → Intro: extending Skip-gram for speed and quality
  • → Hierarchical softmax recap; negative sampling objective (NCE-style)
  • → Subsampling of frequent words (formula and effect)
  • → Learning phrase representations (bigram scoring, merging)
  • → Experiments: word similarity, analogy, phrase analogies
  • → Conclusion: negative sampling + subsampling + phrases = state-of-the-art at the time

Visual — Negative sampling vs full softmax

For each (center word, context word) pair: Full softmax normalizes over all V words (expensive). Negative sampling treats it as binary: one positive (true context) + k negatives (random words from a unigram table). Only k+1 outputs are updated per step.

Skip-gram step:  center = "cat", context = "sat"
Positive pair: (cat, sat) → label 1
Negatives (e.g. k=3): (cat, "the"), (cat, "matrix"), (cat, "run") → label 0
Update: maximize P(sat|cat), minimize P(the|cat), P(matrix|cat), P(run|cat)

How the two papers connect

Paper 1 (Efficient Estimation) introduces the core idea: CBOW and Skip-gram over words, with hierarchical softmax for efficient training. You get one vector per word and linear analogies.

Paper 2 (Phrases and Compositionality) builds directly on Skip-gram: same architecture, but (1) negative sampling replaces hierarchical softmax for faster and often better training, (2) subsampling balances the effect of frequent vs rare words, and (3) phrases get their own tokens and vectors so multi-word expressions (“New_York”, “ice_cream”) are represented properly.

Together they define the standard Word2Vec pipeline: train Skip-gram (or CBOW) with negative sampling and optional subsampling on a word-level or phrase-merged corpus to obtain dense vectors that capture similarity and compositionality. The code in word2vec.c implements both papers (HS + negative sampling, and word2phrase for phrases). Reading Paper 1 first gives you the objective and the tree; Paper 2 gives you the tricks that make it scale and the phrase extension.
